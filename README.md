## 📐 SOLID Principles in C++ — Complete 6-Hour Lecture

A structured, in-depth lecture on the five SOLID design principles, 
written for Software Engineering and Embedded Systems students.

### 📦 What's Inside
- Full theoretical explanation of all 5 SOLID principles
- Real-world analogies for each principle
- ❌ Bad design examples vs ✅ correct C++ implementations
- Mermaid class diagrams (Obsidian-compatible)
- Step-by-step refactoring walkthroughs
- Mini-lab exercise after each principle (10–15 min)
- Final Lab: Smart Home Controller project (50 min)

### 🧠 Principles Covered
| # | Principle | Core Idea |
|---|-----------|-----------|
| S | Single Responsibility | One reason to change |
| O | Open-Closed | Extend without modifying |
| L | Liskov Substitution | Subtypes are substitutable |
| I | Interface Segregation | No fat interfaces |
| D | Dependency Inversion | Depend on abstractions |

### 🛠️ Requirements
- C++17 or later
- Obsidian (for best Markdown + Mermaid rendering)
